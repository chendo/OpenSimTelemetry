OpenSimTelemetry API

REST and SSE endpoints for telemetry data. Base URL: http://localhost:9100

Sections Streaming (SSE) Adapters Custom Metrics Annotations Replay History Sinks Persistence Conversion Sessions (serve mode)

Streaming (SSE)

GET/api/streamSSE

Unified SSE stream multiplexing telemetry frames, adapter status, sink config, and annotation updates. Uses named events: frame, status, sinks, annotations. Preferred over individual stream endpoints (avoids HTTP/1.1 connection limit).

ParamTypeDescription
metric_maskstringComma-separated sections to include (e.g. vehicle,timing)
ratefloatFrames per second (0.01–60, default 60)
formatstringjson (default) or msgpack (base64-encoded MessagePack)
deltaboolEnable delta encoding (default true). Only changed sections are sent; full frame every 60th.

SSE Event Types:

EventDescription
frameTelemetry frame (full or delta). Delta frames have "_delta": true.
statusAdapter status change (JSON with source, connection info).
sinksSink config list updated (JSON array of all sinks).
annotationsAnnotation list updated (JSON array of all annotations).
curl -N "http://localhost:9100/api/stream?rate=10&metric_mask=vehicle,timing"
GET/api/telemetry/streamSSE

Telemetry-only SSE stream. Same query params as /api/stream.

GET/api/status/streamSSE

Adapter status updates SSE stream.

Adapters

GET/api/adapters

List all registered adapters with their detection status.







POST/api/adapters/:name/toggle

Enable or disable an adapter by key name.





Custom Metrics



Submit arbitrary metrics that get merged into telemetry frames. Sticky metrics (no tick) appear in every frame. Tick-specific metrics appear only in the frame with a matching tick number. Custom metrics are included in SSE frame events and GET /api/metrics responses.




POST/api/metrics

Submit custom metrics under a namespace. Metrics are merged into the telemetry frame JSON at the top level using the namespace as the key.


// Sticky metric — merged into every frame
{"namespace": "analysis", "metrics": {"brake_bias": 0.57, "fuel_delta": -0.03}}

// Tick-specific — merged only into frame with tick 42000
{"namespace": "analysis", "metrics": {"optimal_line": true}, "tick": 42000}






GET/api/metrics

Get the latest telemetry frame with custom metrics merged in. Supports metric_mask query param for section filtering.









GET/api/metrics/custom

List all currently stored custom metrics (sticky and tick-specific).









DELETE/api/metrics/custom

Clear all custom metrics (both sticky and tick-specific).






DELETE/api/metrics/custom/:namespace

Clear custom metrics for a specific namespace only.





Annotations



Annotate the telemetry timeline with labeled, colored regions. Annotations can specify ranges by tick number or session time. They appear as colored overlays on the graph widgets and are broadcast to all SSE clients via the annotations event.




POST/api/annotations

Create a new annotation. If id is omitted, one is auto-generated. At least one of the tick or time range fields should be provided.


{
  "title": "Lock-up",
  "color": "#ff6b6b",
  "start_tick": 42000,
  "end_tick": 42300
}

// Or using session time (seconds):
{
  "title": "Pit window",
  "color": "rgba(0, 200, 100, 0.3)",
  "start_time_s": 120.5,
  "end_time_s": 135.0
}






GET/api/annotations

List all current annotations.









DELETE/api/annotations/:id

Delete an annotation by ID. Returns 204 on success, 404 if not found.





Replay




POST/api/replay/upload

Upload an .ibt file to start replay mode. Accepts multipart/form-data with a single file field.


curl -X POST -F "file=@race.ibt" http://localhost:9100/api/replay/upload






GET/api/replay/info

Get current replay/history info (track, car, frame count, playing state, laps).









GET/api/replay/frames

Fetch a range of frames from replay or history buffer.




ParamTypeDescription


startint0-based start frame index


countintNumber of frames to fetch (max 7200)


metric_maskstringComma-separated sections to include


ridstringReplay ID for immutable cache headers








POST/api/replay/control

Control playback: play, pause, seek, set speed.


// Play/pause
{"action": "play"}  |  {"action": "pause"}

// Seek to frame
{"action": "seek", "value": 1000}

// Set speed
{"action": "speed", "value": 2.0}






DELETE/api/replay

Stop and delete the current replay session.





History Buffer




POST/api/history/config

Resize the history buffer.


{"max_duration_secs": 600}  // 60–3600






GET/api/history/aggregate

Compute statistics (min, max, avg, stddev, count) for metrics over a time window.




ParamTypeDescription


metricsstringComma-separated dot-paths (e.g. vehicle.speed,vehicle.rpm)


durationstringTime window: 60s, 5m, 1h (default 60s)

Delete a sink by ID.

Persistence




GET/api/persistence/config

Get current persistence configuration (auto-save, frequency, retention).









POST/api/persistence/config

Update persistence configuration.


{
  "auto_save": true,
  "frequency_hz": 60,
  "max_sessions": 10,
  "max_age_days": 30
}






GET/api/persistence/stats

Get storage statistics (file count, total size).









GET/api/persistence/files

List saved telemetry session files.









POST/api/persistence/load

Load a saved session file into replay mode.


{"filename": "2024-01-15_Spa_McLaren.ost.ndjson.zstd"}






GET/api/persistence/download

Download the current history buffer as ZSTD-compressed NDJSON.






DELETE/api/persistence/files/:name

Delete a saved session file.





Conversion




POST/api/convert/ibt

Convert an .ibt file to ZSTD-compressed NDJSON stream. Accepts multipart/form-data.


curl -X POST -F "file=@race.ibt" http://localhost:9100/api/convert/ibt -o output.ost.ndjson.zstd





Sessions (serve mode)



When the server is started with --serve, session endpoints are available. Users can upload .ibt files to create shareable sessions. Admin endpoints require HTTP Basic auth with OST_ADMIN_USER / OST_ADMIN_PASS credentials.




POST/api/sessions/upload

Upload an .ibt file to create a new session. Returns a shareable URL with an access token. Public — no auth required.


curl -X POST -F "file=@race.ibt" http://localhost:9100/api/sessions/upload

// Response:
{
  "id": "a1b2c3d4e5f6",
  "token": "0123456789abcdef...",
  "url": "/s/a1b2c3d4e5f6?token=0123456789abcdef...",
  "track_name": "Spa-Francorchamps",
  "car_name": "BMW M4 GT3",
  "total_frames": 108000,
  "duration_secs": 1800.0
}






GET/api/sessionsAdmin

List all stored sessions. Requires admin authentication.









DELETE/api/sessions/:idAdmin

Delete a session and its data. Requires admin authentication.






POST/api/sessions/:id/load

Load a session into the replay state. Requires the session's access token as ?token= query parameter.


curl -X POST "http://localhost:9100/api/sessions/abc123/load?token=secret_token"






GET/api/sessions/stats

Get session storage statistics (count, total size, disk cap).









GET/s/:id?token=...

View a session. Validates the access token, loads the session into replay, and serves the telemetry UI. Share this URL to give others access to the session.
